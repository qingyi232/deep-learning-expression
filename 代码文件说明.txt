2.1 gui.py - 图形用户界面主程序
--------------------------------------------------------------------------------
功能：提供PyQt5图形界面，实现用户交互

主要类：
1. VideoThread(QThread)
   - 视频处理线程类
   - 负责摄像头捕获和实时识别
   - 使用多线程避免界面卡顿
   
   主要方法：
   - run(): 运行视频捕获循环
   - stop(): 停止视频捕获
   
   信号：
   - change_pixmap_signal: 发送视频帧
   - update_result_signal: 发送识别结果

2. EmotionRecognitionGUI(QMainWindow)
   - 主窗口类
   - 管理整个图形界面
   
   主要方法：
   - init_ui(): 初始化界面布局
   - create_image_tab(): 创建静态图像识别选项卡
   - create_video_tab(): 创建实时视频识别选项卡
   - load_model(): 加载深度学习模型
   - upload_image(): 上传图片
   - predict_image(): 预测图片表情
   - start_video(): 开始视频识别
   - stop_video(): 停止视频识别
   - update_video_frame(): 更新视频帧显示
   - update_video_result(): 更新识别结果文本

界面布局：
- 标题栏：显示系统名称
- 选项卡：
  - 静态图像识别：上传图片进行识别
  - 实时视频识别：摄像头实时识别
- 状态栏：显示系统状态和设备信息

依赖模块：
- PyQt5: 图形界面框架
- OpenCV: 图像处理
- predictor: 表情预测模块

代码行数：约300行

2.2 model.py - 深度学习模型定义
--------------------------------------------------------------------------------
功能：定义CNN模型架构

主要类：
1. EmotionCNN(nn.Module)
   - 卷积神经网络模型
   - 基于改进的VGG风格架构
   
   网络结构：
   - 输入层：48x48灰度图像
   - 卷积块1：2个卷积层(64通道) + BN + ReLU + MaxPool + Dropout
   - 卷积块2：2个卷积层(128通道) + BN + ReLU + MaxPool + Dropout
   - 卷积块3：2个卷积层(256通道) + BN + ReLU + MaxPool + Dropout
   - 卷积块4：2个卷积层(512通道) + BN + ReLU + MaxPool + Dropout
   - 全连接层1：512*3*3 -> 1024 + BN + ReLU + Dropout
   - 全连接层2：1024 -> 512 + BN + ReLU + Dropout
   - 输出层：512 -> 7 (7种表情)
   
   主要方法：
   - _make_conv_block(): 创建卷积块
   - forward(): 前向传播

主要函数：
- get_model(device): 创建并返回模型实例

模型参数：
- 总参数量：9,939,143
- 可训练参数：9,939,143
- 模型大小：约38MB

技术特点：
- 批归一化(Batch Normalization)：加速训练，提升稳定性
- Dropout正则化：防止过拟合
- ReLU激活函数：非线性变换
- MaxPooling：降维和特征提取

依赖模块：
- torch.nn: 神经网络模块
- config: 配置参数

代码行数：约120行

2.3 predictor.py - 表情预测模块
--------------------------------------------------------------------------------
功能：封装模型预测功能，提供统一接口

主要类：
1. EmotionPredictor
   - 表情识别预测器
   - 封装模型加载、人脸检测、表情预测
   
   主要方法：
   - __init__(): 初始化预测器，加载模型和人脸检测器
   - detect_faces(): 检测图像中的人脸
   - predict_emotion(): 预测单个人脸的表情
   - predict_image(): 预测图像文件中的表情
   - predict_frame(): 预测视频帧中的表情

人脸检测：
- 使用OpenCV Haar Cascade分类器
- 支持多人脸检测
- 三级检测策略（标准->宽松->最宽松）

图像预处理：
- 调整大小：48x48
- 转换为灰度图
- 归一化：mean=0.5, std=0.5
- 转换为Tensor

预测流程：
1. 读取图像（支持中文路径）
2. 检测人脸位置
3. 提取人脸区域
4. 预处理人脸图像
5. 模型推理
6. 返回表情标签和置信度

输出格式：
- emotion_label: 表情标签（中文+英文）
- confidence: 置信度（0-1）
- probabilities: 7种表情的概率分布

依赖模块：
- OpenCV: 人脸检测和图像处理
- PyTorch: 模型推理
- PIL: 图像处理

代码行数：约250行

2.4 dataset.py - 数据集加载和处理
--------------------------------------------------------------------------------
功能：加载FER2013数据集，进行预处理和数据增强

主要类：
1. FER2013Dataset(Dataset)
   - PyTorch数据集类
   - 加载和处理FER2013数据
   
   主要方法：
   - __init__(): 初始化数据集
   - __len__(): 返回数据集大小
   - __getitem__(): 获取单个样本

数据预处理：
- 像素字符串转numpy数组
- reshape为48x48图像
- 转换为PIL Image
- 应用数据增强（训练集）
- 归一化和标准化

数据增强（训练集）：
- 随机水平翻转（p=0.5）
- 随机旋转（±10度）
- 随机亮度调整（±20%）
- 归一化（mean=0.5, std=0.5）

主要函数：
- get_data_loaders(): 创建训练集和测试集的DataLoader

DataLoader配置：
- batch_size: 128（可在config.py修改）
- shuffle: True（训练集）
- num_workers: 4（多线程加载）
- pin_memory: True（GPU加速）

数据集信息：
- 训练集：28,709样本
- 测试集：3,589样本
- 图像大小：48x48灰度图
- 表情类别：7类

依赖模块：
- pandas: 读取CSV
- numpy: 数组处理
- PIL: 图像处理
- torchvision.transforms: 数据增强

代码行数：约150行

2.5 config.py - 系统配置文件
--------------------------------------------------------------------------------
功能：集中管理系统配置参数

主要配置项：

1. 数据集配置
   - DATA_DIR: 数据集目录路径
   - TRAIN_CSV: 训练集CSV路径
   - TEST_CSV: 测试集CSV路径
   - IMAGE_SIZE: 图像大小(48x48)
   - NUM_CLASSES: 表情类别数(7)

2. 模型配置
   - MODEL_SAVE_PATH: 模型保存目录
   - BEST_MODEL_PATH: 最佳模型路径
   - CHECKPOINT_PATH: 检查点路径

3. 训练配置
   - BATCH_SIZE: 批大小(128)
   - LEARNING_RATE: 学习率(0.001)
   - NUM_EPOCHS: 训练轮数(100)
   - EARLY_STOP_PATIENCE: 早停耐心值(15)

4. 表情标签
   - EMOTION_LABELS: 7种表情的中英文标签
     0: 生气 (Angry)
     1: 厌恶 (Disgust)
     2: 恐惧 (Fear)
     3: 开心 (Happy)
     4: 悲伤 (Sad)
     5: 惊讶 (Surprise)
     6: 中性 (Neutral)

使用方式：
import config
print(config.BATCH_SIZE)

代码行数：约50行

3.1 train_fast.py - GPU快速训练脚本
--------------------------------------------------------------------------------
功能：使用GPU加速训练模型

主要特点：
- 强制使用GPU（CUDA）
- 混合精度训练（FP16）
- 自动保存最佳模型
- 早停机制
- 学习率调度
- 训练历史可视化

训练流程：
1. 检查GPU可用性
2. 加载数据集
3. 创建模型
4. 配置优化器和损失函数
5. 训练循环
6. 验证和保存模型
7. 绘制训练曲线

优化策略：
- 混合精度训练：使用GradScaler
- 学习率调度：ReduceLROnPlateau
- 梯度裁剪：max_norm=1.0
- 早停：15轮不提升则停止
- cudnn.benchmark：自动优化

训练参数：
- 优化器：Adam
- 学习率：0.001
- 批大小：128
- 最大轮数：100
- 早停耐心值：15

输出文件：
- models/best_model.pth：最佳模型
- models/training_history.png：训练曲线

训练时间：
- GPU：约2-5小时
- CPU：约20-50小时（不推荐）

代码行数：约200行

3.2 train.py - 标准训练脚本
--------------------------------------------------------------------------------
功能：标准训练流程，支持CPU和GPU

与train_fast.py的区别：
